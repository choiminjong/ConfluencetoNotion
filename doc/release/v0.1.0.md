# v0.1.0 (2026-02-22)

최초 릴리즈. Confluence 페이지를 Markdown으로 변환하고, Notion API 블록 JSON으로 2차 변환 후 Notion DB에 업로드하는 파이프라인.

## 프로젝트 구조

```
confluencetoNotion/
├── convert.py                   # 변환 파이프라인 (Step 1 + Step 2)
├── confluence_to_markdown.py    # Step 1: Confluence -> Markdown
├── markdown_to_notion.py        # Step 2: Markdown -> Notion JSON
├── upload.py                    # Notion DB 업로드
├── list_databases.py            # Notion DB 목록 조회 유틸
├── .env                         # 환경변수 (CONFLUENCE_URL, PAT, NOTION_TOKEN, DB_ID)
├── pyproject.toml               # uv 의존성 관리
├── package/
│   ├── confluence-markdown-exporter/   # 외부 패키지 (editable)
│   └── notion-markdown/               # 외부 패키지 (editable)
├── doc/                         # 문서
├── output/                      # 변환 결과물
└── release/                     # 릴리즈 기록
```

## 주요 기능

- **convert.py**: Confluence 페이지 ID 기반으로 Markdown + 첨부파일 + Notion JSON 자동 변환
- **upload.py**: 변환된 데이터를 Notion DB에 페이지 생성 및 이미지 업로드
- **list_databases.py**: Notion Integration이 접근 가능한 DB 목록 조회

## 수정된 이슈 목록

### 1. 테이블 중첩/이미지 처리 (confluence_to_markdown.py)

- **파일**: `confluence_to_markdown.py` — `TableOverride` 클래스
- **문제**: 패키지 원본의 테이블 변환이 Confluence Server의 중첩 테이블, 테이블 내 이미지, 테이블 내 줄바꿈을 제대로 처리하지 못함
- **해결**: `TableOverride` 클래스로 패키지의 `convert_table` 메서드를 오버라이드
  - 중첩 테이블: HTML 원본 유지
  - 테이블 내 이미지: 텍스트 링크(`📎 filename`)로 대체, 테이블 하단에 이미지 블록 배치
  - 셀 내 줄바꿈: `<br>` 구분자 유지

### 2. 테이블 셀 줄바꿈 누락 (confluence_to_markdown.py)

- **파일**: `confluence_to_markdown.py` — `TableOverride.cell_to_text()`
- **문제**: 패키지 `convert_p`가 `td` 내부에서 `\n`을 모두 제거하여 셀 내용이 한 줄로 합쳐짐
- **해결**: `cell.decode_contents()`로 `<td>` 래퍼 없이 내부 HTML만 추출하여 `td` 컨텍스트를 우회

### 3. 인접 bold 마커 깨짐 (confluence_to_markdown.py)

- **파일**: `confluence_to_markdown.py` — `TableOverride.cell_to_text()`
- **문제**: `****`(인접한 bold 종료+시작)가 Notion 파서에서 `***` + `*`로 잘못 분리됨
- **해결**: `"****"` → `"**<br>**"`로 치환하여 bold 구분자를 명확히 분리

### 4. 테이블 하단 이미지 블록 누락 (confluence_to_markdown.py)

- **파일**: `confluence_to_markdown.py` — `TableOverride.convert()`
- **문제**: 추출된 이미지를 `"\n".join()`으로 연결하여 이미지 간 빈 줄 없음 → notion-markdown이 마지막 이미지만 image 블록으로 인식
- **해결**: `"\n\n".join(extracted_images)`로 변경하여 각 이미지가 독립 paragraph로 인식

### 5. Confluence 링크 제거 시 ordered list 오인식 (markdown_to_notion.py)

- **파일**: `markdown_to_notion.py` — `strip_confluence_links()`
- **문제**: `[01. text](confluence_url)` → `01. text` 변환 후, `01.`이 ordered list 시작으로 파싱되어 빈 bullet + nested numbered list 구조 생성 (Notion에서 "리스트" 표시)
- **해결**: 링크 텍스트가 `숫자.`로 시작하면 `.`을 `\.`로 escape 처리하여 ordered list 인식 방지

### 6. 리스트 내 heading 패턴 (markdown_to_notion.py)

- **파일**: `markdown_to_notion.py` — `convert_list_headings()`
- **문제**: `- ### text` (bullet + heading) 조합이 Markdown/Notion에서 지원 안 됨 → 빈 bullet + heading children 구조로 잘못 변환
- **해결**: `- ### text` → `- **text**` (bullet + bold)로 변환하여 강조는 유지하면서 정상 bullet 구조 보장

### 7. 첨부파일 다운로드 모드 선택 (confluence_to_markdown.py)

- **파일**: `confluence_to_markdown.py` — `download_attachments()`
- **문제**: 모든 첨부파일을 다운로드하면 Markdown에서 참조하지 않는 불필요한 파일도 포함됨
- **해결**: `download_all_attachments` 플래그 추가. `False`면 Markdown에서 참조하는 파일만, `True`면 전체 다운로드

### 8. Notion DB 속성 자동 생성 (upload.py)

- **파일**: `upload.py` — `ensure_properties()`
- **문제**: 새 DB에 필수 속성(Confluence ID, Space, Updated, Parent Title)이 없으면 업로드 실패
- **해결**: DB 스키마 조회 후 누락 속성 자동 추가
