# v0.3.1 (2026-02-23)

빈 `<li>` 래퍼로 인한 이중 불릿(`- -`) 변환 오류 수정, 페이지 제목 중복 제거, 변환 설정 파라미터화, `uv run` 스크립트 엔트리 추가, Confluence Tabs 매크로 → Notion Toggle 변환, rich_text 자동 분할, 에러 리포트 시스템 추가.

## 주요 변경사항

### 1. 변환 설정 파라미터화 (run_convert.py, confluence_to_markdown.py)

- **파일**: `run_convert.py` — `Pipeline`, `pipeline/confluence_to_markdown.py` — `ConfluenceExporter`
- **문제**: `page_breadcrumbs` 설정이 `ConfluenceExporter.__init__`에 하드코딩되어 있어 진입점(`main()`)에서 설정을 확인하거나 변경할 수 없음
- **해결**: `include_document_title`, `page_breadcrumbs` 파라미터를 `Pipeline` → `ConfluenceExporter`로 전달하는 구조로 변경. 모든 변환 설정이 `main()`에서 한눈에 보임

  ```python
  Pipeline(
      include_document_title=False,  # Notion 페이지 제목과 중복 방지
      page_breadcrumbs=False,        # 상단 경로 미포함
  )
  ```

### 2. 페이지 제목 중복 제거 (`include_document_title=False`)

- **문제**: Confluence 페이지 제목이 MD 본문 H1 + `meta.json` title + Notion 페이지 제목으로 3중 표시
- **해결**: `include_document_title=False` 기본 설정으로 MD 본문에 패키지가 추가하는 제목 H1을 미포함. Notion 업로드 시 `meta.json`의 title만 페이지 제목으로 사용

### 3. Confluence Tabs 매크로 → Notion Toggle 변환 (converter_overrides.py)

- **파일**: `pipeline/converter_overrides.py` (신규), `pipeline/confluence_to_markdown.py`
- **문제**: Confluence Tabs 플러그인(`data-macro-name="auitabs"`)에 대한 핸들러가 없어 탭 전체 내용이 중첩 blockquote로 변환됨. 이후 단계에서 하나의 callout(45,000자+)으로 합쳐져 Notion API 제한 초과
- **해결**: `converter_overrides.py`에 Tabs 오버라이딩 핸들러 추가. 각 탭을 `<details><summary>탭제목</summary>내용</details>` 구조로 변환하고, `notion-markdown` 패키지가 이를 Notion Toggle 블록으로 자동 변환
- **설계**: 패키지 원본 수정 없이 `convert_div`를 래핑하는 오버라이딩 방식. 기존 `TableOverride`도 같은 파일로 이동하여 오버라이딩 코드를 한 곳에서 관리

### 4. 오버라이딩 코드 파일 분리 (converter_overrides.py)

- **파일**: `pipeline/converter_overrides.py` (신규)
- **문제**: `confluence_to_markdown.py`에 오버라이딩 코드(TableOverride 등)가 ~100줄 차지하여 비즈니스 로직 가독성 저하
- **해결**: 모든 오버라이딩 코드를 `converter_overrides.py`로 분리. `apply_overrides(converter_cls)` 한 줄로 일괄 적용. 향후 새 매크로 핸들러 추가 시 이 파일에만 작업

### 5. Notion API rich_text 자동 분할 (upload.py)

- **파일**: `upload/upload.py`
- **문제**: Notion API는 `text.content` 최대 2,000자, `rich_text` 배열 최대 100개 제한. 이를 초과하는 콘텐츠가 있으면 업로드 실패 (400 에러)
- **해결**: `sanitize_blocks()` 메서드 추가. 모든 블록을 재귀 순회하며 `rich_text`를 2,000자 이하로 자동 분할. 줄바꿈 > 공백 > 강제 절단 순으로 경계를 탐색하여 데이터 유실 없이 분할. `table_row`의 cells도 처리

### 6. 에러 리포트 시스템 (convert_errors.json, upload_errors.json)

- **파일**: `pipeline/confluence_to_markdown.py`, `pipeline/markdown_to_notion.py`, `upload/upload.py`, `run_convert.py`, `run_upload.py`
- **문제**: 에러 발생 시 `print`만 하고 넘어가서, 콘솔 로그가 사라지면 실패 원인 추적 불가
- **해결**: 각 단계에서 에러를 수집하여 `convert_errors.json` / `upload_errors.json`으로 저장. page_id, title, step, error_type, message, timestamp 기록. 실행 완료 후 콘솔에 요약 출력

## 수정된 이슈 목록

### 7. 빈 `<li>` 래퍼로 이중 불릿 생성 (confluence.py)

- **파일**: `package/confluence-markdown-exporter` — `Converter.convert_li()`
- **문제**: Confluence가 중첩 리스트를 텍스트 없는 빈 `<li>` 안에 감싸는 HTML을 생성. `markdownify`가 이를 그대로 변환하여 `- - 항목` (이중 불릿) 패턴 발생

  ```html
  <!-- Confluence가 생성하는 HTML -->
  <ul>
    <li>              <!-- 텍스트 없는 빈 <li> -->
      <ul>
        <li>프로젝트 선택</li>
        <li>템플릿 내용 입력</li>
      </ul>
    </li>
  </ul>
  ```

  ```markdown
  # 변환 결과 (수정 전)
  - - 프로젝트 선택       ← 이중 불릿
    - 템플릿 내용 입력

  # 변환 결과 (수정 후)
  - 프로젝트 선택
  - 템플릿 내용 입력
  ```

- **해결**: `convert_li()`에서 `<li>`의 직접 텍스트(`NavigableString`)가 없고 자식이 `<ul>/<ol>`만인 경우, 빈 불릿을 생성하지 않고 중첩 리스트를 상위 레벨로 승격

### 8. `uv run` 스크립트 엔트리 추가 (pyproject.toml)

- **파일**: `pyproject.toml`, `run_convert.py`
- **문제**: `uv sync` 후 `python run_convert.py`로 실행하면 시스템 Python을 사용하여 모듈을 찾지 못함
- **해결**: `[project.scripts]`에 `run-convert` 엔트리 추가, `[build-system]` 설정으로 패키지화. `uv run run-convert` 또는 `uv run python run_convert.py`로 venv 활성화 없이 실행 가능
