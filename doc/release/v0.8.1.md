# v0.8.1 — 청크 텍스트 정제 (Notion 마크다운 기호 제거)

**날짜**: 2026-02-27

---

## 주요 변경사항

### 1. 임베딩 노이즈 제거

Neo4j Content 노드의 `chunk` 텍스트에서 Notion 마크다운 기호를 전면 제거하여 임베딩 품질을 개선하고 토큰 비용을 절감한다.

| 기호 유형 | Before | After |
|---|---|---|
| 불릿 마커 | `- 항목 텍스트` | `항목 텍스트` |
| 넘버링 | `1. 항목 텍스트` | `항목 텍스트` |
| 체크박스 | `- [x] 완료 항목` | `완료 항목` |
| 코드블록 마커 | `` ```bash\ncommand\n``` `` | `command` |
| 헤딩 마커 | `### 소제목` | `소제목` |
| 테이블 파이프 | `\| key \| value \|` | `key: value` |
| 참조 태그 | `[하위 페이지: 설정]` | (제거) |
| 임베드/링크 태그 | `[임베드: https://...]` | (제거) |
| 경로 접두사 | `[경로 \| 경로]\n` | `경로 > 경로\n` |
| 소제목 인라인 | `## 소제목` | `소제목` |
| 들여쓰기 | depth 기반 공백 | (제거) |

### 2. clean_chunk_text() 안전망 함수

`chunker.py`에 청크 최종 텍스트 정제 함수를 추가하여 잔여 기호를 처리한다.

| 처리 대상 | 동작 |
|---|---|
| 3개 이상 연속 줄바꿈 | `\n\n`으로 축소 |
| 해시태그 (`#태그`) | 제거 (Topics 노드로 별도 관리) |
| 대괄호 (`[버튼]`) | `버튼`으로 변환 |
| 앞뒤 공백 | 정리 |

### 3. 테이블 변환 개선

기존 파이프 구분(`| cell | cell |`)에서 의미 기반 변환으로 변경:

- 2열 테이블: `key: value` 형태
- 3열 이상: 쉼표(`, `) 구분
- 헤더 행이 있으면 별도 출력 후 데이터 행 시작

## Before / After 예시

**Before (v0.8.0)**:

```
[사내 메일 연결 (PC) | 윈도우 (최초연결)]
### a.  Outlook 실행
### b.  전자 메일 주소 입력 후 [연결] 버튼 클릭
- PC에서 Outlook 최초 설정이 아닌 경우...
| 프로필 이름 | 프로필을 구분하기 위한 이름 |
[임베드: https://www.notion.so/12345]
#Outlook #아웃룩
```

**After (v0.8.1)**:

```
사내 메일 연결 (PC) > 윈도우 (최초연결)
a. Outlook 실행
b. 전자 메일 주소 입력 후 연결 버튼 클릭
PC에서 Outlook 최초 설정이 아닌 경우...
프로필 이름: 프로필을 구분하기 위한 이름
```

## 변경 파일

| 파일 | 내용 |
|---|---|
| `graphrag/scraper/section_parser.py` | `_block_to_text()`: 불릿·넘버링·체크박스·코드마커·헤딩마커·들여쓰기 제거, 참조/임베드/링크 태그 빈 문자열 반환. `_parse_table()`: 파이프 기호 제거, key-value 또는 쉼표 구분 변환 |
| `graphrag/graph/chunker.py` | `clean_chunk_text()` 함수 추가, 소제목 `## ` 마커 제거, 경로 접두사 `[|]` → `>` 변환 |

## 비즈니스 로직 영향

- **기존 Neo4j 데이터 재구축 필요**: 정제된 텍스트를 반영하려면 다음 명령을 순서대로 실행
  1. `python -m graphrag.step1_scraper.run` (섹션 재생성)
  2. `python -m graphrag.step2_graph.run` (그래프 재구축)
  3. `python -m graphrag.step3_embedding.run` (임베딩 재생성)
- Phase 1 변환/업로드 로직 변경 없음
- API 엔드포인트 변경 없음
