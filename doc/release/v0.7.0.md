# v0.7.0 — 프로젝트 구조 재편 + Phase 2 GraphRAG 파이프라인

**날짜**: 2026-02-23

---

## 주요 변경사항

### 1. 프로젝트 구조 재편 (B안: Step 단위 분리)

기존 프로젝트 루트의 `run_convert.py`, `run_upload.py`를 각 모듈 폴더로 이동하여 Step 단위로 독립 실행할 수 있도록 구조를 재편했다.

| 변경 전 | 변경 후 |
|---|---|
| `run_convert.py` | `pipeline/run.py` |
| `run_upload.py` | `upload/run.py` |

실행 방법이 통일되었다:

```bash
python -m pipeline.run        # Phase 1 STEP 1
python -m upload.run           # Phase 1 STEP 2
python -m graphrag.step1_scraper.run   # Phase 2 STEP 3
python -m graphrag.step2_graph.run    # Phase 2 STEP 4
python -m graphrag.step3_embedding.run # Phase 2 STEP 5
python -m graphrag.step4_rag.run      # Phase 2 STEP 6
python -m graphrag.step5_web.run      # Phase 2 STEP 7
```

### 2. 경로 안전화

모든 `run.py`에서 `Path("./output")` 같은 CWD 의존적 상대 경로를 `Path(__file__).resolve().parent.parent` 기준의 절대 경로로 변경했다. 어느 디렉토리에서 실행해도 항상 프로젝트 루트의 `output/`과 `.env`를 정확히 찾는다.

### 3. Phase 2: Notion → Neo4j GraphRAG 파이프라인

`graphrag/` 폴더를 신규 생성하여 4단계 파이프라인을 구성했다:

#### STEP 3: Notion 데이터 추출 (`graphrag/scraper/`)
- Notion API로 DB 전체 페이지 속성 + 블록 콘텐츠 추출
- Excel로 저장 (`output/graphrag/Pages_*.xlsx`)

#### STEP 4: Neo4j 그래프 구축 (`graphrag/graph/`)
- Excel → Neo4j 그래프 (Page, Content, Space, Domain, Topic 노드)
- 관계: HAS_CHUNK, BELONGS_TO, IN_SPACE, HAS_TOPIC, CHILD_OF
- Recursive 텍스트 청킹 (500자, 100자 오버랩)

#### STEP 5: GraphRAG 검색 파이프라인 (`graphrag/retriever/`)
- Azure OpenAI 임베딩 생성 + 벡터 인덱스 구축
- 3종 Retriever: VectorRetriever, VectorCypherRetriever, Text2CypherRetriever
- ToolsRetriever로 LLM이 자동 선택
- GraphRAG 2종 프롬프트 (목록/요약)

#### STEP 6: 웹 시각화 서버 (`graphrag/web/`)
- FastAPI + vis-network.js 기반 웹 UI
- GET /graph: 그래프 시각화 데이터
- POST /query: GraphRAG 질의 처리
- GET /health: 서버 상태 확인

### 4. 그래프 스키마

```
Space ──IN_SPACE──► Page ──HAS_CHUNK──► Content
                     │
                     ├──BELONGS_TO──► Domain
                     ├──HAS_TOPIC──► Topic
                     └──CHILD_OF──► Page (Parent)
```

### 5. 의존성 추가

`pyproject.toml`에 Phase 2 관련 패키지 추가:

- `neo4j`, `neo4j-graphrag`: Neo4j 드라이버 + GraphRAG
- `anthropic[bedrock]`: AWS Bedrock Claude LLM
- `openai`, `httpx`: Azure OpenAI 임베딩
- `pandas`, `openpyxl`: Excel 처리
- `fastapi`, `uvicorn[standard]`: 웹 서버

### 6. Notion API 중첩 제한 대응

Confluence 문서에서 발생하는 중첩 구조를 Notion API 제한에 맞게 자동 보정한다.

| 수정 | 파일 | 기능 |
|------|------|------|
| `strip_blockquote_in_details` | `md_preprocessor.py` | `> <details>` 내부 `>` 접두사 제거 |
| `flatten_nested_blockquotes` | `md_preprocessor.py` | `> >` 이중 인용을 `>` 단일 인용으로 변환 |
| `flatten_nested_quotes` | `notion_postprocessor.py` | `quote > quote` 구조를 paragraph로 평탄화 |
| `flatten_nested_quotes` 호출 | `upload.py` | 업로드 시에도 적용 (기존 JSON 재업로드 시) |

### 7. 업로드 실패 페이지 관리

- **빈 페이지 자동 아카이브**: `append_blocks` 실패 시 생성된 빈 Notion 페이지를 자동 아카이브하고 에러를 정확하게 보고
- **실패 페이지 재업로드**: `upload/run_retry.py` 추가 — `upload_errors.json` 기반으로 실패 페이지만 선별하여 재업로드

```bash
python -m upload.run_retry                  # 대화형 폴더 선택
python -m upload.run_retry 폴더명           # 폴더 직접 지정
```
