# v0.9.0 — 테이블 이미지 링크 + rowspan/colspan 오버라이드 정리

**날짜**: 2026-02-28

---

## 주요 변경사항

### 1. 테이블 이미지 내부 링크 (`link_table_images`)

테이블 셀 안의 `📎 파일명` 자리표시자를 실제 이미지 블록으로 이동하는 Notion 앵커 링크로 자동 교체한다.

| 항목 | 설명 |
|---|---|
| 동작 시점 | `upload_page()` 완료 직후 자동 실행 |
| 대상 | `table_row` 블록 셀 내 `📎 filename.png` 텍스트 |
| 결과 | 해당 텍스트에 이미지 블록 위치로 이동하는 클릭 가능한 링크 추가 |
| 실패 처리 | 링크 실패 시 경고만 출력, 페이지 업로드 자체는 정상 유지 |

#### 내부 구현

- `_get_all_children(page_id)`: Notion API 페이지네이션 처리로 전체 블록 트리를 재귀 수집
- `_extract_image_filename(block)`: `file`, `external`, `file_upload` 3가지 image 타입에서 파일명 추출
- `link_table_images(page_id)`: 이미지 블록 ID 매핑 후 `table_row` 셀의 `rich_text`에 링크 패치

### 2. rowspan/colspan 셀 내용 복제 — 패키지 오버라이드 분리

패키지 원본 `table_converter.py`를 직접 수정하던 방식에서 오버라이드 방식으로 전환했다.

| 항목 | 변경 전 | 변경 후 |
|---|---|---|
| 수정 위치 | 패키지 `pad()` 직접 수정 | `converter_overrides.py`의 `_pad_with_content()` |
| 패키지 상태 | 변경됨 (규칙 위반) | 원본 유지 |
| 기능 동작 | 동일 (셀 내용 복제) | 동일 (셀 내용 복제) |

Notion은 테이블 셀 병합(merge)을 지원하지 않으므로, `rowspan`/`colspan`으로 병합된 셀의 내용을 빈 셀 대신 원본 내용으로 복제하여 가독성을 확보한다.

### 3. 중첩 테이블 인라인 변환

`_inner_table_to_inline()` 메서드로 중첩 테이블을 `[ 열1 │ 열2 │ 열3 ]` 형태의 정렬된 인라인 텍스트로 변환한다.

- CJK/한글 문자는 2칸, 그 외 1칸으로 계산하여 `_display_width()` 기반 패딩 적용
- 구분자 `│`(U+2502)를 사용하여 마크다운 파이프 테이블 구문과 충돌 방지

### 4. callout 변환 버그 수정

#### 들여쓰기된 Alert 블록 매칭 실패 수정

Confluence 패널이 중첩 구조 안에서 변환되면 `  > [!WARNING]` 처럼 앞에 공백이 붙는다.
기존 `_ALERT_RE` 정규식이 `^>` (줄 시작 바로 `>`)만 매칭하여 들여쓰기된 경우 callout 변환이 스킵됐다.

| 항목 | 변경 전 | 변경 후 |
|---|---|---|
| Alert 매칭 | `^> \[!WARNING\]` | `^\s*> \[!WARNING\]` |
| body line strip | `^>\s?` | `^\s*>\s?` |

#### callout 내 단일 heading 제거 안 되는 문제 수정

`convert_alerts_to_callouts()` 내부에서 heading 마크를 제거할 때 `r"^###?\s*"` 정규식이 `##`/`###`만 처리하고 단일 `#`을 제거하지 않았다.

| 변경 전 | 변경 후 |
|---|---|
| `r"^###?\s*"` (##, ### 만) | `r"^#{1,6}\s*"` (모든 heading 레벨) |

### 5. output 폴더 자동 생성

`pipeline/run.py`에서 `output/` 폴더가 없으면 에러 대신 자동 생성하도록 변경했다.

## 변경 파일

| 파일 | 변경 내용 |
|---|---|
| `pipeline/converter_overrides.py` | `_pad_with_content()` 추가, `pad()` import 제거 |
| `pipeline/md_preprocessor.py` | `_ALERT_RE` 들여쓰기 대응, heading strip `#{1,6}` 확장, body strip 들여쓰기 대응 |
| `upload/upload.py` | `_get_all_children()`, `_extract_image_filename()`, `link_table_images()` 추가 |
| `pipeline/run.py` | output 폴더 자동 생성 |
| `package/.../table_converter.py` | 원상복구 (패키지 원본) |

## 검증

- `_pad_with_content()` vs 패키지 `pad()` 비교 테스트: colspan/rowspan 셀 내용 복제 검증 통과
- `TableOverride.convert()` E2E 테스트: 실제 테이블 HTML → 마크다운 변환 정상 확인
- `link_table_images()`: 업로드 후 비파괴적 실행 (실패 시 경고만, 페이지 영향 없음)
- 페이지 1630678230: callout 내 `# heading` 제거 확인
- 페이지 1647922386: 들여쓰기된 `> [!WARNING]` → `<aside>⚠️ ...` 정상 변환 확인
