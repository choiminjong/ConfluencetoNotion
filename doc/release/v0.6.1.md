# v0.6.1 — 미지원 매크로 감지, columnLayout 순차 변환, 업로드 속도 개선

날짜: 2026-02-23

---

## 미지원 매크로 자동 감지

- `converter_overrides.py`: 오버라이드 / 패키지 핸들러 모두에 없는 매크로를 자동 감지
  - `_KNOWN_PACKAGE_MACROS` 상수로 패키지 지원 매크로 목록 관리
  - 미지원 매크로 발견 시 콘솔 경고 출력 및 `unsupported_macros` 리스트에 수집
  - `reset_unsupported_macros()` 함수로 실행 간 초기화
- `confluence_to_markdown.py`: 변환 완료 후 `unsupported_macros.json` 자동 생성
  - 매크로별 그룹핑으로 어떤 매크로가 어떤 페이지에서 발견됐는지 정리
  - Converter에 `_current_page_id` 전달하여 페이지 식별

## columnLayout 순차 변환

- `converter_overrides.py`: `convert_column_layout` 오버라이드 추가
  - 기존: 다단 레이아웃을 `<table>`로 변환 → 내부 이미지/HTML 깨짐
  - 변경: 각 컬럼을 순서대로 일렬 배치, 구분선(`---`)으로 시각 분리
  - Notion은 다단 레이아웃 미지원이므로 데이터 보존 우선 전략
  - 내부 이미지, 링크 등 기존 마크다운 변환 파이프라인 정상 통과

## API 요청 간격 단축

- `upload.py`: `REQUEST_INTERVAL`을 0.35s → 0.20s로 단축
  - Notion API Rate Limit(3 req/s)에 가까워지지만, 기존 429 자동 재시도 로직(최대 5회, `Retry-After` 헤더 기반 대기)이 안전장치 역할
  - 파일 업로드가 많은 페이지에서 체감 속도 약 40% 향상
  - 데이터 유실 위험 없음 (429는 일시 거부일 뿐, 재시도로 정상 완료)
