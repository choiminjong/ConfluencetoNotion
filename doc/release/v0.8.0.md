# v0.8.0 — 섹션 기반 청킹 + 동적 스키마 그래프 + 웹 시각화 대폭 개선

**날짜**: 2026-02-25

---

## 주요 변경사항

### 1. 섹션 기반 하이브리드 청킹

기존 recursive 텍스트 분할(500자 고정)에서 **heading 기반 섹션 구조 청킹**으로 전면 교체했다.

| 항목 | v0.7.0 | v0.8.0 |
|---|---|---|
| 청킹 단위 | 500자 고정 분할 | 섹션(heading) 기반 의미 단위 |
| 컨텍스트 | 없음 | `heading_path` 계층 경로 주입 |
| 최소 크기 | 없음 | 500자 미만 인접 섹션 자동 병합 |
| 순서 관계 | 없음 | `NEXT_CHUNK` 관계로 청크 순서 보존 |
| 미디어 노이즈 | 포함 | `📎 파일명`, `[이미지: ...]` 자동 제거 |

#### heading_path 컨텍스트 주입

각 청크 텍스트 앞에 섹션 경로가 자동 삽입되어 임베딩 품질이 향상된다:

```
[섹션: 1. 개요 > 1.1 설치 방법]
Docker를 사용하여 설치하는 방법을 안내합니다...
```

#### 인접 소형 섹션 병합 (Hybrid Merging)

500자 미만의 짧은 섹션은 동일 depth의 인접 섹션과 병합하여 의미 있는 크기의 청크를 생성한다.

### 2. 미디어 노이즈 제거

Confluence에서 변환된 문서의 미디어 참조 텍스트를 청킹 시 자동 제거한다.

- `📎 Radio_buttons.png` → 제거
- `[이미지: screenshot of radio buttons]` → 제거
- `[비디오: demo.mp4]` → 제거

`graphrag/scraper/section_parser.py`에서 `_clean_media_noise()` 함수로 구현.

### 3. Notion 데이터 추출 JSON 전환

STEP 3(Notion 데이터 추출)의 출력 형식을 Excel에서 **구조화된 JSON**으로 전환했다.

| 항목 | v0.7.0 | v0.8.0 |
|---|---|---|
| 출력 형식 | `Pages_*.xlsx` | `pages_*.json` |
| 본문 구조 | 플레인 텍스트 | `sections[]` (heading_path + content_type + text) |
| 스키마 정보 | 없음 | `db_schema` 포함 |
| 원본 블록 | 미포함 | `blocks` 전체 보존 |

#### section_parser.py (신규)

Notion API 블록 27개 타입을 모두 지원하는 구조화 파서. heading 계층을 재귀적으로 탐색하여 섹션 트리를 구성한다.

### 4. 스키마 기반 동적 그래프 구축

STEP 4(Neo4j 그래프 구축)에서 Notion DB 스키마를 읽어 **select/multi_select 필드를 자동으로 노드와 관계로 매핑**한다. 고정 필드(Space, Domain, Topic)만 지원하던 구조에서 동적 스키마 기반으로 확장했다.

```
DB 스키마에 "Content Type" (select) 필드가 있으면:
  → ContentType 노드 자동 생성
  → Page ─HAS_CONTENTTYPE─► ContentType 관계 자동 생성
```

### 5. 웹 시각화 대폭 개선

`graphrag/web/static/index.html`을 전면 재구성하여 IVGraph 수준의 기능을 구현했다.

#### 그래프 뷰

| 기능 | 설명 |
|---|---|
| 2D/3D 토글 | vis-network.js(2D) / 3d-force-graph(3D) 전환 |
| 전체 화면 렌더링 | CSS 수정으로 그래프 캔버스가 패널 전체 영역 사용 |
| 노드 필터 칩 | 노드 타입별 표시/숨김 (Space, Status는 기본 숨김) |
| 허브 노드 자동 dim | 연결이 많은 노드를 투명하게 처리 (Domain 제외) |
| 물리 엔진 제어 | 안정화 후 자동 비활성화 (드래그/팬 snap-back 방지) |
| 노드 간격 최적화 | springLength 200, centralGravity 0.2로 넓은 배치 |
| 그래프 커스터마이징 | 중력, 노드 크기, 엣지 라벨, 허브 dim 슬라이더 |

#### 3D 시각화

| 기능 | 설명 |
|---|---|
| 엣지 색상 | 출발 노드 타입 색상 기반 |
| 엣지 굵기 | 1.5px (기존 0.5px 대비 3배) |
| 방향 입자 | 선을 따라 흐르는 파티클로 방향성 시각화 |
| 투명도 | 0.6 불투명도로 깊이감 표현 |

#### Smart Search & Filter

| 기능 | 설명 |
|---|---|
| 실시간 검색 | 노드 제목 검색, 3글자 이상 시 자동 하이라이트 |
| 타입 필터 | 노드 타입별 칩 토글 |
| Progressive Disclosure | Page 클릭 시 Content 청크 동적 로딩 |

#### 노드 정보 패널

| 기능 | 설명 |
|---|---|
| 상세 속성 | 노드의 모든 Neo4j 속성 표시 |
| 연결 노드 목록 | 해당 노드와 연결된 노드 리스트 (클릭 탐색) |
| AI 유사 노드 | `/similar/{page_id}` 엔드포인트 (RAG 연결 시) |

#### 분석 대시보드

| 기능 | 설명 |
|---|---|
| 통계 카드 | 노드 수, 엣지 수, 밀도, 평균 연결 수 |
| Topic 분포 차트 | Chart.js 가로 막대 그래프 |

### 6. API 엔드포인트 확장

`graphrag/web/routers/graph.py`에 추가된 엔드포인트:

| 엔드포인트 | 설명 |
|---|---|
| `GET /graph?mode=semantic` | 시맨틱 뷰 (Page + 속성 노드만) |
| `GET /graph?mode=full` | 전체 뷰 (Content 청크 포함) |
| `GET /graph/expand/{page_id}` | Progressive Disclosure: 특정 Page의 Content 청크 |
| `GET /graph/similar/{page_id}` | AI 유사 노드 추천 |
| `GET /graph/analytics` | 그래프 통계 |

### 7. RAG 서비스 Lazy Loading

`graphrag/web/services/rag_service.py`에서 BedrockLLM과 AzureOpenAIEmbeddings를 `initialize()` 함수 내부로 이동하여 **자격증명 없이도 그래프 시각화가 동작**하도록 변경했다.

## 스키마 변경

### NEXT_CHUNK 관계 추가

```
Content ──NEXT_CHUNK──► Content
```

섹션 기반 청킹 시 청크 간 순서를 보존하는 관계가 추가되었다. 기존 HAS_CHUNK 관계는 유지된다.

### Content 노드 속성 추가

| 속성 | 타입 | 설명 |
|---|---|---|
| `heading_path` | string | 섹션 계층 경로 (예: `개요 > 설치 방법`) |
| `content_type` | string | 콘텐츠 유형 (procedure, reference, mixed 등) |
