# v0.5.0 (2026-02-23)

이미지 누락 수정(볼드 래핑, 헤딩 내부, 콜아웃/토글 내부), 비디오 첨부파일 다운로드/업로드 지원, 콜아웃 내 링크 파싱, 깨진 이메일 링크 자동 수정.

## 주요 변경사항

### 1. 볼드 래핑 이미지 복원 (md_preprocessor.py)

- **파일**: `pipeline/md_preprocessor.py`
- **문제**: Confluence 마크다운에서 이미지가 `**![](img.png)**` 형태로 볼드 래핑됨. `notion-markdown` 패키지가 이를 볼드 텍스트로 인식하여 이미지 블록이 생성되지 않고 텍스트로 표시
- **해결**: `strip_bold_images()` 함수 추가. `**![alt](url)**` -> `![alt](url)` 변환. `preprocess()` 파이프라인 초반에 실행

### 2. 헤딩 내부 이미지 추출 (md_preprocessor.py)

- **파일**: `pipeline/md_preprocessor.py`
- **문제**: `# ![](img.png)` 또는 `# **![](img.png)**` 형태로 헤딩 안에 이미지만 있는 경우, 헤딩 텍스트로 인식되어 이미지 블록 미생성
- **해결**: `extract_heading_images()` 함수 추가. 이미지만 포함된 헤딩을 제거하고 이미지 블록만 남김

### 3. 콜아웃/토글 내부 이미지 → children 블록 변환 (notion_postprocessor.py)

- **파일**: `pipeline/notion_postprocessor.py`
- **문제**: `<aside>` (콜아웃) 및 `<details>` (토글) 내부의 `![](img)` 마크다운이 plain text로 처리되어 이미지가 누락
- **해결**:
  - `parse_callout_content()`: 콜아웃 rich_text에서 `![alt](url)` 패턴을 감지하여 텍스트에서 제거 후 callout children에 image 블록 추가
  - `extract_images_from_text_blocks()`: 토글/콜아웃 children paragraph의 인라인 이미지를 별도 image 블록으로 분리

### 4. 로컬 미디어 추출 재귀화 (notion_postprocessor.py)

- **파일**: `pipeline/notion_postprocessor.py`
- **문제**: `extract_local_images()`가 top-level 블록만 탐색하여 callout/toggle children 내부의 이미지를 누락
- **해결**: `extract_local_media()`로 확장. children을 재귀적으로 탐색하고 video 블록도 함께 추출

### 5. 비디오 첨부파일 다운로드 (confluence_to_markdown.py)

- **파일**: `pipeline/confluence_to_markdown.py`
- **문제**: 비디오 파일(`[name.mp4](path)`)이 링크 구문이라 `extract_referenced_files()`가 감지하지 못함. 비디오 경로도 `..\..\attachments\.mp4` 형태로 깨져 있음
- **해결**:
  - `rewrite_video_paths()`: 깨진 비디오 링크 경로를 파일명으로 교체 (`[name.mp4](..path..)` -> `[name.mp4](name.mp4)`)
  - `extract_referenced_files()` 확장: 이미지(`![](...)`) 외에 비디오 링크(`[name.mp4](...)`)도 매칭하여 다운로드 대상에 포함

### 6. 비디오 → Notion video 블록 변환 (md_preprocessor.py, notion_postprocessor.py)

- **파일**: `pipeline/md_preprocessor.py`, `pipeline/notion_postprocessor.py`
- **문제**: Notion에 비디오 블록 타입이 존재하지만, 마크다운 링크가 그대로 텍스트/paragraph로 변환됨
- **해결**:
  - `convert_video_links()`: 비디오 링크 구문을 이미지 구문으로 변환하여 블록 생성 유도 (`[name.mp4](path)` -> `![video](name.mp4)`)
  - `convert_image_to_video()`: URL 확장자가 비디오인 image 블록을 Notion `video` 블록으로 타입 변환

### 7. 비디오 블록 업로드 지원 (block_utils.py, upload.py)

- **파일**: `upload/block_utils.py`, `upload/upload.py`
- **문제**: `transform_blocks()`와 `filter_invalid_images()`가 image 블록만 처리. video 블록 미지원
- **해결**:
  - `transform_blocks()`: image와 video 블록 모두 `file_upload` 참조로 변환
  - `filter_invalid_media()` (기존 `filter_invalid_images` 대체): image와 video 블록 모두 유효성 검사
  - `upload_page()`: `local_images` 목록의 비디오 파일도 Notion에 업로드

### 8. 깨진 이메일 링크 자동 수정 (md_preprocessor.py)

- **파일**: `pipeline/md_preprocessor.py`
- **문제**: Confluence 마크다운에서 이메일이 `office-it@[nexon.co.kr](http://nexon.co.kr)` 형태로 깨짐 (도메인만 링크, 이메일 전체가 아님). 6개 페이지에서 발견
- **해결**: `fix_broken_email_links()` 함수 추가. `text@[domain](http://domain)` -> `[text@domain](mailto:text@domain)` 변환. 모든 이메일 형식에 범용 적용

### 9. 콜아웃 내부 링크 파싱 (notion_postprocessor.py)

- **파일**: `pipeline/notion_postprocessor.py`
- **문제**: 콜아웃(`<aside>`) 내부의 마크다운 링크(`[text](url)`)가 plain text로 처리되어 Notion에서 클릭 불가
- **해결**: `parse_callout_content()` 함수 추가. 기존 `parse_bold_rich_text()`를 대체하여 콜아웃 내 모든 마크다운 구문 처리:
  - `**bold**` -> `annotations.bold: true`
  - `[text](url)` -> `rich_text.link.url`
  - `**[text](url)**` -> bold + link 동시 적용
  - `![](img)` -> children image 블록
