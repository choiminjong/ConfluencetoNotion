# v0.4.0 (2026-02-23)

Confluence 이모티콘 변환, YAML 프론트매터 → Notion DB 태그 매핑, 콜아웃 볼드/줄바꿈 정상 처리, SKIP 페이지 트리 복원, 비즈니스/유틸리티 로직 분리 리팩토링.

## 주요 변경사항

### 1. Confluence 이모티콘 SVG → 유니코드 이모지 변환 (converter_overrides.py, confluence_to_markdown.py)

- **파일**: `pipeline/converter_overrides.py`, `pipeline/confluence_to_markdown.py`
- **문제**: Confluence 이모티콘 SVG 이미지(`check.svg`, `error.svg` 등)가 상대 URL로 마크다운에 포함됨. Notion API에서 "Invalid image url" 에러가 발생하여 전체 블록 추가가 실패하고 페이지 본문이 비어버림
- **해결**: 두 단계로 처리
  - `TableOverride.extract_images()`: 테이블 내 `<img>` 태그에서 이모티콘 SVG 경로 감지 시 `CONFLUENCE_EMOTICON_MAP`으로 유니코드 이모지 치환
  - `replace_emoticon_markdown()`: 마크다운 본문의 인라인 이모티콘 이미지 참조(`![](/path/emoticons/check.svg)`)를 정규식으로 감지하여 이모지 치환
- **매핑**: `check.svg` → ✅, `error.svg` → ❌, `warning.svg` → ⚠️ 등 24개. 미등록 이모티콘은 ▪️ (fallback) + WARN 로그

### 2. 잘못된 이미지 URL 필터 안전장치 (block_utils.py, upload.py)

- **파일**: `upload/block_utils.py`, `upload/upload.py`
- **문제**: 이모티콘 변환을 거쳤음에도 간혹 남아있는 non-HTTP 이미지 URL이 Notion API 에러를 유발
- **해결**: `filter_invalid_images()` 함수 추가. 업로드 전 `external.url`이 `http`로 시작하지 않는 이미지 블록을 재귀적으로 제거. WARN 로그 출력

### 3. YAML 프론트매터 → Notion DB Topics 태그 매핑 (md_preprocessor.py, upload.py)

- **파일**: `pipeline/md_preprocessor.py`, `upload/upload.py`
- **문제**: `confluence-markdown-exporter`가 Confluence 라벨을 YAML 프론트매터로 추출(`tags: - "#지라_플러그인"`). `notion-markdown` 패키지가 이를 plain text로 처리하여 Notion 페이지 본문에 `tags:`, `"#지라_플러그인"` 텍스트가 노출
- **해결**:
  - `strip_frontmatter()`: 프론트매터를 제거하고 tags 추출. `\uXXXX` 유니코드 이스케이프 자동 디코딩
  - `upload.py`: 추출된 tags를 Notion DB `Topics` multi_select 속성으로 매핑. 태그 없는 페이지는 빈 값

### 4. 콜아웃 볼드 + 줄바꿈 정상 처리 (notion_postprocessor.py, md_preprocessor.py)

- **파일**: `pipeline/notion_postprocessor.py`, `pipeline/md_preprocessor.py`
- **문제**: `> [!NOTE]` 블록을 `<aside>`로 변환 시 `**bold**` 마크다운이 literal text로 표시되고, 여러 줄이 한 줄로 합쳐짐
- **해결**:
  - `convert_alerts_to_callouts()`: 줄바꿈을 `{{BR}}` 플레이스홀더로 보존
  - `fix_callout_bold()`: Notion JSON 후처리에서 `{{BR}}` → `\n` 복원 + `**bold**` → `rich_text` annotation(`"bold": true`) 분할 변환

### 5. 빈 헤딩 자동 제거 (md_preprocessor.py)

- **파일**: `pipeline/md_preprocessor.py`
- **문제**: `include_document_title=False` 설정 시 패키지가 제목 텍스트는 비우지만 빈 `# ` 마커를 남겨둠. Notion에서 "제목1" 플레이스홀더로 표시
- **해결**: `strip_empty_headings()` 추가. 텍스트 없는 헤딩(`# `, `## ` 등)을 정규식으로 제거

### 6. SKIP 페이지 트리 구조 복원 (confluence_to_markdown.py)

- **파일**: `pipeline/confluence_to_markdown.py`
- **문제**: 빈 컨테이너 페이지(예: "필드", "이슈관리")를 SKIP할 때 parent-child 관계 정보도 함께 사라짐. `index.json` 트리가 flat 구조가 되어 `Parent Title` 필드가 빈 값으로 업로드
- **해결**:
  - `skipped_parents` 딕셔너리로 SKIP 페이지의 `page_id → parent_id` 매핑 보존
  - `_resolve_parent()`: SKIP된 부모 체인을 따라가며 가장 가까운 내보낸 조상을 찾음
  - `update_children()`에서 각 페이지의 `parent_id`를 re-parenting하여 트리 복원

### 7. Page Properties (details) 매크로 오버라이드 (converter_overrides.py)

- **파일**: `pipeline/converter_overrides.py`
- **문제**: `confluence-markdown-exporter`가 `details` 매크로(Page Properties)를 YAML frontmatter로 추출. `sanitize_key()`가 한글 키를 지원하지 않아 "플러그인 명" 같은 컬럼명 유실
- **해결**: `_convert_page_properties()` 오버라이드 추가. frontmatter 추출 대신 이미 `TableOverride`가 변환한 마크다운 테이블을 본문에 그대로 유지

## 리팩토링

### 8. 비즈니스 / 유틸리티 로직 분리

- **파일**: `pipeline/md_preprocessor.py` (신규), `pipeline/notion_postprocessor.py` (신규), `pipeline/markdown_to_notion.py` (슬림화)
- **문제**: `markdown_to_notion.py` (338줄)에 비즈니스 오케스트레이션과 마크다운 전처리 + Notion JSON 후처리 유틸리티가 혼재. 새 변환 규칙 추가 시 파일이 비대해지고 비즈니스 흐름 파악 어려움
- **해결**:
  - `pipeline/md_preprocessor.py`: 마크다운 전처리 함수 모음 (stateless 순수 함수). `strip_frontmatter()`, `preprocess()` 등 7개 함수 + 파이프라인
  - `pipeline/notion_postprocessor.py`: Notion JSON 후처리 함수 모음. `fix_callout_bold()`, `fix_toggle_children()`, `postprocess()` 등 5개 함수 + 파이프라인
  - `pipeline/markdown_to_notion.py`: 비즈니스 오케스트레이션만 유지 (~120줄). `convert_page()`, `convert_all()`, `_save_error_report()`
