## 주요 변경사항

### 1. 환경변수 전면 `.env` 관리

모든 설정을 `.env`에서 읽도록 통일하였다. 코드 내 기본값(fallback) 제거.

- `CONFLUENCE_PAGE_IDS` — 변환 대상 페이지 ID를 `.env`에서 관리 (쉼표 구분)
- `NOTION_API_URL`, `NOTION_API_PATH`, `NOTION_API_VERSION` — Notion API 설정을 필수 환경변수로 전환 (사내 프록시 대응)
- 비즈니스 로직 클래스(`NotionUploader`, `ConfluenceExporter`)는 환경변수를 직접 읽지 않고 파라미터로 받음

### 2. 진입점 클래스화

- `run_convert.py` — `pipeline/convert.py`의 `Pipeline` 클래스를 통합. `pipeline/convert.py` 삭제
- `run_upload.py` — `UploadRunner` 클래스로 리팩토링. 환경변수 검증을 `__init__`에서 수행

### 3. Notion DB 필드 변경

- **제거**: `Confluence ID` (number) — `Source URL`로 대체
- **추가**: `Source URL` (url) — Confluence 원본 페이지 전체 URL

### 4. `meta.json`에 `source_url` 추가

`pipeline/confluence_to_markdown.py`에서 `meta.json` 생성 시 Confluence 원본 URL을 자동으로 포함:

```json
{
  "source_url": "https://confluence.example.com/pages/viewpage.action?pageId=1462521351"
}
```

### 5. Notion API 에러 핸들링

`upload/upload.py`와 `utils/get_bot_databases.py`에 `_check_response()` 메서드 추가:

- 401: 토큰 무효 안내
- 403: 접근 권한 없음 안내
- 404: DB 미발견 / Integration 미연결 안내
- 400: 상세 에러 메시지 출력
- traceback 대신 사용자 친화적 메시지 출력 후 종료

### 6. 유틸리티 리네임 + 클래스화

- `utils/list_databases.py` → `utils/get_bot_databases.py`
- 함수 기반 → `DatabaseFinder` 클래스로 리팩토링
- Search API에서 DB 제목이 비어있으면 `GET /databases/{id}`로 직접 조회하여 제목 보완

### 7. 이미지 업로드 버그 수정

- `upload/upload.py`의 이미지 업로드 시 `self.VERSION` 참조 오류 수정 → `self.headers["Notion-Version"]` 사용

### 8. `.env.example` / `.gitignore` 추가

- `.env.example` — 모든 환경변수의 설명과 기본값을 포함한 템플릿
- `.gitignore` — `.env`, `.venv/`, `output/`, `__pycache__/` 등 제외 설정
