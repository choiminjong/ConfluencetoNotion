# v0.2.0 (2026-02-22)

프로젝트 구조 재편 및 Notion DB 필드 확장. 변환과 업로드를 독립 진입점으로 분리하고, 온톨로지/지식그래프 구현을 위한 DB 스키마를 설계하였다.

## 프로젝트 구조 변경

기존 루트에 흩어져 있던 스크립트를 모듈 단위로 정리하였다.

```
# 이전 (v0.1.0)                        # 이후 (v0.2.0)
confluencetoNotion/                     confluencetoNotion/
├── main.py                             ├── run_convert.py          ← Step 1 진입점
├── convert.py                          ├── run_upload.py           ← Step 2 진입점 (대화형)
├── confluence_to_markdown.py           ├── pipeline/
├── markdown_to_notion.py               │   ├── convert.py
├── upload.py                           │   ├── confluence_to_markdown.py
├── list_databases.py                   │   └── markdown_to_notion.py
                                        ├── upload/
                                        │   └── upload.py
                                        └── utils/
                                            └── list_databases.py
```

## 주요 변경사항

### 1. 진입점 분리 (`main.py` → `run_convert.py` + `run_upload.py`)

- `main.py` 삭제, 변환과 업로드를 별도 스크립트로 분리
- 각 진입점에 필수 환경변수 검증 로직 추가
- `run_upload.py`에 대화형 폴더 선택 및 Domain 입력 기능 추가

### 2. Notion DB 필드 확장

필수 속성(`REQUIRED_PROPERTIES`)에 운영 관리를 위한 필드를 추가하였다:

- **Status** (select): 문서 상태 관리 (Active / Deprecated / Draft, 기본값 Active)
- **Topics** (multi_select): 문서 주제 태그 (온톨로지 추출 시 활용)

커스텀 속성(`CUSTOM_PROPERTIES`)으로 사용자가 직접 지정하는 필드를 도입하였다:

- **Domain** (select): 문서의 주제 영역 (Jira, Confluence, CI/CD 등)

### 3. 모듈 패키지화

- `pipeline/`: 변환 파이프라인 모듈 (`convert.py`, `confluence_to_markdown.py`, `markdown_to_notion.py`)
- `upload/`: Notion 업로드 모듈 (`upload.py`)
- `utils/`: 유틸리티 (`list_databases.py`)
- 각 모듈에 `__init__.py` 추가, 상대 import 적용

### 4. 환경변수 검증

- `run_convert.py`: `CONFLUENCE_URL`, `CONFLUENCE_PAT` 누락 시 에러 메시지 출력 후 종료
- `run_upload.py`: `NOTION_TOKEN`, `NOTION_DATABASE_ID` 누락 시 에러 메시지 출력 후 종료
